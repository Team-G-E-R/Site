SpriteGen — фронтенд с правильным мок/HTTP слоем авторизации

Главное:
- По умолчанию проект работает БЕЗ бэкенда (VITE_BACKEND=mock) — регистрация/вход через localStorage.
- Для подключения настоящего API есть режим VITE_BACKEND=http (использует fetch + VITE_API_BASE).
- UI НЕ знает, мок это или настоящий бэкенд: UI общается только с apiRegister/apiLogin/apiMe/apiLogout.

Быстрый старт (mock, только фронт):
1) Установи Node.js LTS.
2) Двойной клик: start-front.bat
3) Открой: http://localhost:5173

Что потестить (mock режим):
A) Открой сайт → нажми "Войти / Регистрация" → Регистрация:
   - введи Имя, Email, Пароль (>=6) → "Продолжить"
   - модалка закроется, сверху появится "Привет, <имя>"
B) Нажми "Кабинет" → проверь отображение имени/email/даты.
C) Нажми "Выйти" → снова кнопка "Войти / Регистрация".
D) Войди тем же email/паролем → снова "Привет, ..."
E) Обнови страницу (F5) → ты всё ещё залогинен (сессия хранится в localStorage).
F) В кабинете нажми "Dev: users" → должен показаться список mock-пользователей.

Где хранятся mock-пользователи:
- DevTools (F12) → Application → Local Storage → http://localhost:5173
  * mock_users_v1 (список пользователей)
  * mock_session_v1 (id текущей сессии)

Режим HTTP (для будущего подключения к реальному бэкенду):
- Включается VITE_BACKEND=http и VITE_API_BASE=http://... (см. start-http-demo.bat).
- В архиве есть server-sample — ДЕМО бэкенд, чтобы проверить, что HTTP-режим работает.
